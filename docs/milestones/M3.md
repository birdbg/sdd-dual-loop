# M3：可恢复的变更驱动双环

## 定位

M3 在不改变双环核心节点的前提下，补充执行反馈跨环回流、人工规格变更、检查点恢复和运行内规格版本管理。

## 双环结构保持不变

```text
Purpose
→ Brainstorming
→ Plan
→ Verify
→ Dev
→ Test
→ Refactor
→ Archive
```

## M3 只解决四个问题

- 反馈路由
- 人工 Change Spec
- 检查点恢复
- Spec、Plan、Verify 运行内版本管理

## 反馈路由规则

```text
code_error → development
test_error → testing
plan_omission → planning
spec_ambiguous → change_spec
purpose_conflict → purpose
blocked → archive
```

## Change Spec 约束

- 只能由人工批准
- 不允许 LLM 自行批准规格变更
- 规格变更后必须回到 Brainstorming
- 必须重新生成 Plan
- 必须重新 Verify
- 不允许直接返回 Dev

## 恢复约束

- 已完成节点不得重复执行
- 已创建分支不得重复创建
- 已成功完成的阶段产物不得被无条件覆盖
- 恢复前必须检查目标仓库、基线提交和工作分支
- 状态不一致时停止
- 不自动 reset、merge 或 push

## M3 明确不做

- 多 Agent
- 多模型路由
- Prompt Registry
- Skill Registry
- Web 管理端
- 企业权限
- 多租户
- Git Worktree
- 自动合并
- 自动推送
- 自动批准 Change Spec
- 多技术栈
- 数据库状态中心
- 分布式调度

- RunContext 增加版本和恢复字段
