# Brainstorming

## 方案 A：路由直接遍历用户列表

在 `main.py` 中调用 `list_users()` 并查找 ID。改动最少，但路由承担数据查询逻辑，职责混杂，后续测试和复用较差。

## 方案 B：用户存储提供单用户查询函数

在 `users.py` 增加一个按 ID 返回 `User | None` 的小函数；路由负责参数约束和把“未找到”映射为 HTTP 404。职责清楚，改动仍然很小，也便于单元测试。

## 方案 C：引入 Repository 层

定义仓储接口和内存实现，再由路由依赖注入。扩展性较好，但对只有两个只读端点的 M1 示例明显过度设计。

## 选择

选择方案 B。它在保持 M1 范围最小的同时，将数据查询和 HTTP 行为分开；不引入新层级或依赖。
